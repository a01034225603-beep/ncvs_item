## 스킬 1 : 플랜 모드
새 기능 개발 시작 전 항상 먼저 실행.
코드 작성하지 말고 계획만 보여줘.
계획 항목:
1. 생성/수정할 파일 목록
2. 각 파일에서 작업할 내용
3. 구현 순서
4. 예상 위험요소
5. 테스트 케이스 목록

---

## 스킬 2 : TDD
구현 전에 테스트 코드 먼저 작성.
순서: 테스트 작성 → 실행(실패 확인) → 구현 → 실행(통과 확인)
규칙:
- 실제 소켓 mock 사용 (실제 BACS 없이 테스트 가능하게)
- 각 테스트는 독립적으로 실행 가능해야 함
- 한국어 주석 필수

---

## 스킬 3 : 수술식 구현
파일 전체 재작성 금지.
지정한 함수/클래스만 작성.
다른 파일 건드리지 마.

---

## 스킬 4 : 디아그노즈
에러 발생 시 추측으로 바로 수정 금지.
반드시 아래 순서로:
1. 재현 — 에러 재현 최소 조건 정의
2. 가설 — 원인 가설 2~3개 나열 (확률 높은 순)
3. 확인 — 각 가설 확인 방법 제시
4. 수정 — 확인된 원인만 해당 부분 수정

---

## 스킬 5 : 새 컨텍스트 리뷰
핵심 파일 완성 후 실행.
구현 과정 모르는 상태에서 코드만 보고 리뷰.
리뷰 항목:
1. async 패턴 맞는지
2. 패킷 파싱 로직 오류 없는지
3. 엣지케이스 누락 없는지
4. 테스트 커버리지 누락 없는지
5. 한국어 주석 있는지

## UDP 프로토콜 (확정)
포트: 7788
요청 패킷 (9바이트): 01 10 05 00 00 00 12 00 00
응답 패킷 (17바이트): 01 01 0D 00 00 00 92 08 00 + [8바이트]
타임아웃: 5초
판정:
- 응답 옴 + 헤더 일치 → ONLINE
- 타임아웃 → OFFLINE
- 헤더 불일치 → OFFLINE
