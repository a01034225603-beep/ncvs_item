End-to-End Flow

click-by-click trace
session walkthrough

Seed & Login User

관리자가 한 번만 python -m app.cli.seed admin admin으로 계정 + 데모 디바이스 5대를 생성한다. 이후 사용자는 /login에서 자격을 제출한다.

FRONT POST /api/auth/login → BACK bcrypt.checkpw · create_access_token(HS256) → FRONT localStorage.setItem("ncvs_token") · router.push("/devices")

Devices Dashboard Browser

App Router 클라이언트 컴포넌트가 마운트되면 토큰을 확인하고 /devices와 /devices/health를 호출한다. 이후 3초 간격으로 헬스만 다시 폴링한다.

FRONT GET /api/devices + /api/devices/health → BACK device_repo.list_all / health_repo.list_all → FRONT setInterval(3000) · HealthBadge 색상 갱신

Refresh Health Optional

대기 시간 없이 즉시 핑을 보내고 싶으면 Refresh Health 버튼이 강제 sweep을 트리거한다. 백엔드는 모든 enabled 장비에 UDP CONNECT_REQ를 동시(concurrency=50) 전송한다.

FRONT POST /api/health/refresh → BACK HealthCheckService.run_once(db) → WIRE UDP Connect REQ × N · timeout 3s → upsert device_health

Start Cross-Test User → System

2개 이상 장비를 선택하고 시작하면 permutations(ids, 2)으로 방향성 pair가 만들어진다. 세션은 queued 상태로 DB에 박히고, scheduler에 submit(session_id)가 호출된다.

FRONT POST /api/tests {device_ids} → BACK SessionService.create_session · pair_repo.bulk_insert_pending → BACK CrossTestScheduler.submit(id) · 100ms dispatch tick

Pair Dispatch Loop Background

100ms마다 스케줄러가 pick_next_dispatchable을 호출해 (src, dst) 양쪽 모두 잠겨있지 않은 pair를 찾는다. 잠금 획득 → PairRunner로 위임 → UDP cross-test 프로토콜 실행(현재는 stub) → 결과 기록 후 락 해제.

BACK DeviceLocker.try_acquire_pair(src, dst) → asyncio.Lock × 2 → BACK PairRunner.run → CrossTestProtocol.run_pair(timeout=70s) → BACK pair_repo.mark_result · upsert_latest · session.increment_counters

Live Progress Browser

/tests/[id]가 3초 간격으로 세션 상태를 폴링한다. done_pairs / total_pairs 비율로 진행률 바가, ok_pairs · fail_pairs가 카운터로 노출. running 동안만 Cancel 버튼이 보인다.

FRONT GET /api/tests/{id} every 3s · React.use(params) → FRONT TestProgress · status 전환 (queued → running → completed)

Matrix View Browser

세션이 끝나면 (혹은 진행 중이라도) /matrix에서 pair_latest_result의 N×N grid를 확인한다. 같은 (src, dst)에 대한 가장 최근 결과 1건만 표시되므로 색이 항상 최신 상태를 반영한다.

FRONT GET /api/pair-matrix?device_ids=… → BACK SELECT pair_latest_result WHERE src/dst IN ids → FRONT <Matrix /> · 녹/적/회색 셀 + hover tested_at

UI Redesign Proposal

control-room density

현재 Tailwind 스캐폴드는 동작하지만 화이트보드 같은 인상이 강합니다. "제어실 디스플레이"의 정체성으로 다시 잡으면 정보 밀도, 신뢰감, 실시간성이 동시에 올라갑니다. 네 페이지 모두 동일한 다크 베이스 + 시안/네온그린 강조 + 모노스페이스 라벨 시스템으로 통일합니다.

suggested roll-out
  1. PASS·1
    디자인 토큰 + 글로벌 스타일

    위 globals.css 교체 + Google Fonts 3종 임포트. layout.tsx에 font-sans 적용. 페이지 0개 수정으로 즉시 다크 전환.

  2. PASS·2
    공통 컴포넌트 introducing

    Panel (corner-cut frame), LabelMono, StatPill, TopBar 컴포넌트 추가. 한 번 만들면 4개 페이지가 동일 디자인 언어로 자동 정렬.

  3. PASS·3
    페이지 단위 리스킨

    /login → /devices(카드+스파크라인) → /tests/[id](KPI 타일+라이브 로그) → /matrix(글로우+drill-down) 순. 각 페이지 독립 PR/커밋 가능.

  4. PASS·4
    백엔드 보강(선택)

    스파크라인용 GET /devices/health/history?bucket=5m, 라이브 로그용 GET /tests/{id}/events?after_id=… 또는 SSE. UI 먼저 mock 데이터로 만들고 백엔드는 나중에 붙여도 됨.
